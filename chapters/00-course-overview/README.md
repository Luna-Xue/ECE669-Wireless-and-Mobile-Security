# Chapter 0: Course Overview and Learning Path

> Why this course sits between traditional wireless/mobile security and AI security — and how
> the Build / Break / Fix path takes you through it.

**ECE 669 · [Course README](../../README.md) · [Syllabus](../../SYLLABUS.md)**

## What this chapter covers

- **Course goals** — read wireless/mobile systems securely, use AI to *assist* engineering
  without letting it *become* the authority, and reason about AI failures as **wireless
  consequences**.
- **Required vs. optional background** — see the [course README](../../README.md#who-this-is-for--required-vs-optional-background). Short version: basic
  cybersecurity *or* AI/ML, some wireless/networking interest, and programming are required;
  GPU/SDR/5G-testbed/O-RAN are **not**.
- **How traditional wireless security and AI cybersecurity connect** — AI enters the
  engineering workflow (Ch. 2), which creates new risks (Ch. 3), which land on real wireless
  control surfaces (Ch. 8).
- **The Build / Break / Fix learning model** — build an explainer agent (Ch. 5), break it in
  the sandbox (Ch. 6), and add safety gates before control (Ch. 7).

## Orientation "lab"

No attack lab here — just set up your environment and run the sandbox smoke test:

```bash
open ../ai-ran-diagnosis/ui/index.html   # zero-install demo of the explainer agent
```

Then read the **[Sandbox Safety Boundary](../../README.md#sandbox-safety-boundary)** — it
governs every later chapter.
