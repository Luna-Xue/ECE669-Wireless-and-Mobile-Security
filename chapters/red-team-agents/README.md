# Ch.4 — Red Team: Attack a Peer's Agent

> Learn AI-security systematically, then weaponize it against another team's agent.

**Module of [ECE 669 — Wireless and Mobile Security](../../SYLLABUS.md) · Weeks 11–13**

> 🚧 **Status:** landing page + outline. Attack harness / rubric coming.

## What this module is about

You receive another team's agent (from [Ch.3](../train-an-agent/)) and try to break it.

- **AI-security primer** — jailbreaks; prompt injection (direct & indirect); PII /
  training-data / system-prompt extraction; the attack taxonomy; how to measure
  **attack-success-rate (ASR)**. You have already met these as *instances* in Ch.2/Ch.3 —
  here they become a system.
- **Attack** — black-box first, then white-box. Quantify ASR; document every exploit.

The explainer's injection samples
([`../ai-ran-diagnosis/samples/`](../ai-ran-diagnosis/samples/) — obvious vs. stealth) are the
entry-level target to warm up on.

## Deliverable → hands back to Ch.5

An **exploit report** with measured ASR, returned to the owning team, who then hardens in
[Ch.5](../blue-team-capstone/).

> **Graded on process, not on who wins** — see the [syllabus](../../SYLLABUS.md).

## Weekly

- **Wk 11** — AI-security primer + ASR methodology
- **Wk 12** — attack: black-box
- **Wk 13** — attack: white-box; quantify ASR; report
