# Chapter 8: Wireless / PLS / ISAC Consequences

> Why a fooled explainer matters: trace each AI failure to a concrete wireless, physical-layer
> secrecy, or sensing consequence — including a *simulated* robot-car.

**ECE 669 · [Course README](../../README.md) · [Syllabus](../../SYLLABUS.md)**

> 🚧 **Status:** landing page + outline. All scenarios are **simulated** (no real control).

## Topics

- **Unsafe power or beam recommendation** — an AI-suggested setting that would degrade or
  expose the link.
- **Scheduling or mobility risk** — bad handover / resource decisions.
- **Artificial-noise and secrecy risk** — physical-layer-security settings that undermine
  confidentiality.
- **Sensing confidence and blockage uncertainty** — acting on low-confidence ISAC evidence.
- **ISAC confused robot-car scenario** — a simulated cyber-physical case where confused
  sensing would produce an unsafe action, and where a safety gate ([Ch. 7](../07-fix-safety-gates/))
  prevents it.

## Lab — consequence scenarios

Run the simulated scenarios and connect each Ch. 6 attack to the consequence it *would* have
caused, and each Ch. 7 gate to the consequence it prevents. Simulation only — nothing is
actuated.
