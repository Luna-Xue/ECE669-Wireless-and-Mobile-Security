# Chapter 2: AI-Assisted Engineering for Security Workflows

> How AI actually speeds up security engineering — and the discipline that keeps AI output
> from being mistaken for evidence.

**ECE 669 · [Course README](../../README.md) · [Syllabus](../../SYLLABUS.md)**

> 🚧 **Status:** landing page + outline. AI-assisted-artifact lab coming.

## Topics

- **AI for scripts, tests, debugging, and log analysis** — using an assistant to draft
  parsers, generate test cases, explain a stack trace, or summarize a log window.
- **LLM-based system explanation** — turning raw evidence into a human-readable account of
  *what happened*.
- **Human verification of AI-generated artifacts** — how to check generated code and
  explanations before trusting them.
- **Why AI output is not evidence by itself** — a fluent explanation is a *hypothesis*; it
  must be grounded in verifiable evidence (the theme that carries into Ch. 4–7).

## Lab

**AI-assisted artifact + verification** — use an AI assistant to draft a log parser or test
against **synthetic** data, then verify it and find where it is wrong or unsupported.
